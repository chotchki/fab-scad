Workflow tabs — app-wide, editor-first

Full-width tabs across the top of the window, each owning the whole app. The 3D view is persistent and changes per stage; Connectors is no longer top-level — it drills down inside Parts. Two entry points: desktop opens on the picker, web presupplies the model and lands you in the editor.

03

Two feedback channels

At-a-glance on the tabs; detail on the bottom bar. An invalidated tab carries an amber dot; the tab that's actively rendering shows motion (a spinner). The bottom status bar — traditional, app-wide — spells out exactly what's running. The reactive standard: no modal spinners, no rebuild buttons; work just happens and the chrome reports it.

05

Scope — now vs later

Top tab bar + per-tab body now

A single app-wide Tab resource drives a full-width bar; each tab swaps the side panel's contents and what the 3D view renders. Replaces the derived PanelMode.

Code editor in Model now

egui's multiline text edit + a re-render-on-change hook. The unsliced preview is just the current render path pointed at the edited buffer.

Model = one tab, files as inner tabs now

Settled: a single Model tab. Opened files are inner tabs (VS Code-style) with a ＋ that reopens the folder browser. Reuses the existing FileList/SwitchFile machinery — the active file already is the downstream driver.

Wizard affordances later

Free tabs now — no step-gating, validation, or completion checks. The DAG staleness dots (amber, on stale downstream tabs) are the only guidance in v1.
